CTXMAMComplianceDelegate

Extends Protocol:
NSObject
Declared In:
CTXMAMCompliance.h

Introduction

This protocol provides callback with NSError objects with CTXMAMComplianceError codes and localized message strings.

Discussion

App should implement any callback methods that it wants to have custom behavior for, and the implementations should adhere to the documented guidelines.

- (BOOL) handleAdminLockAppSecurityActionForError:(NSError*)error;
Parameters

Return Value

Discussion

handleEDPComplianceViolationForError:

To handle compliance violation when the device falls below the minimum compliance requirements of encryption. 

- (BOOL) handleEDPComplianceViolationForError:(NSError*)error;
Parameters
error

CTXMAMComplianceError error code and localized message in NSLocalizedDescriptionKey of user info.

Return Value

YES/true if app handles this; otherwise NO/false.

Discussion

If error code is

- CTXMAMCompliance_Violation_EDP_BlockApp: Handle it by requiring the user to quit the app and suggesting to re-launch the app after fixing the compliance violation based on the suggestion mentioned in the user info message.

- CTXMAMCompliance_Violation_EDP_WarnUser: Handle it by warning user of the compliance violation and risks of leaking app container data when device is lost. For example, by prompting an obtrusive alert view.

- CTXMAMCompliance_Violation_EDP_InformUser: Handle it by unobtrusively informing user of the compliance violation and risks of leaking app container data when device is lost.
